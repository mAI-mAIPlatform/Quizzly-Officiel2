# Accessibilite ♿

Objectif: rendre Quizzly praticable par tous les joueurs/etudiants.

## Couleurs et contrastes

- Respecter WCAG AA pour texte et boutons.
- Mode contraste eleve toggle dans Reglages.

## Clavier et lecteurs d'ecran

- Navigation tablineaire sur desktop.
- Roles ARIA pour boutons, listes, progression.
- Labels explicites pour les formulaires (matiere, niveau).

## Animations

- Respecter preference `prefers-reduced-motion`.
- Offrir une option "reduire animations" dans Reglages.

## Audio et feedback

- Sons systeme optionnels, volume reglable.
- Sous-titres pour toute video ou voix off.

## Taille et tap targets

- Cibles tactiles >= 44x44 px.
- Espacement suffisant pour eviter les tap errors.

## Tests

- Parcourir l'onboarding avec VoiceOver/Narrator.
- Verifier les couleurs en mode daltonien (protanopie/deuteranopie).

## Checkpoints a respecter

- Pas de texte uniquement en couleur.
- Focus visible en permanence.
